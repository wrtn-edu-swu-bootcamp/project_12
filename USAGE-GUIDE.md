## 📝 방법 1: 웹에서 바로 사용 (가장 쉬움)

### 1단계: 파일 열기 (5초)

```
index.html 파일을 더블클릭
```

웹 브라우저가 자동으로 열립니다.

### 2단계: 정보 입력 (30초)

**필수 입력 항목 5가지:**

#### ① 데이터 크기

```
행 개수: 1000000  (100만 행)
열 개수: 50
```

**💡 팁:**
- 엑셀: 맨 아래 행 번호 확인
- Python: `df.shape` 결과의 첫 번째 숫자 (행), 두 번째 숫자 (열)
- SQL: `SELECT COUNT(*) FROM table`

#### ② 분석 방법

```
예: 랜덤 포레스트 선택
```

**💡 팁:**
- ✅ 표시: 정확한 예측 (추천)
- ⚠️ 표시: 추정 예측 (참고용)

#### ③ 사용 툴

```
Python / R / SQL / Excel 중 선택
```

**💡 팁:**
- Python이 가장 정확합니다
- Excel은 대용량 데이터에 부적합

#### ④ 하드웨어 성능

```
저사양 / 중간 / 고사양 / 최고사양
```

**내 컴퓨터는 어디?**

| 성능 | CPU | RAM | 저장장치 | 예시 |
|------|-----|-----|----------|------|
| 저사양 | 2코어 | 4GB | HDD | 오래된 노트북 |
| 중간 | 4코어 | 8-16GB | SSD | 일반 노트북/데스크탑 |
| 고사양 | 8코어 | 32GB | SSD | 워크스테이션 |
| 최고사양 | 16코어+ | 64GB+ | SSD+GPU | 서버급 |

**💡 어떻게 확인하나요?**
- Windows: 작업 관리자 → 성능 탭
- Mac: 활성 상태 보기 (Activity Monitor)

#### ⑤ 데이터 타입 비율 (합계 100%)

```
수치형: 70%
범주형: 20%
텍스트: 10%
```

**데이터 타입이 뭐예요?**

| 타입 | 설명 | 예시 |
|------|------|------|
| 수치형 | 숫자 | 나이, 가격, 온도 |
| 범주형 | 카테고리 | 성별, 지역, 등급 |
| 텍스트 | 문자 | 리뷰, 설명, 주소 |

**💡 모르겠어요!**  
→ 기본값 (70% / 20% / 10%)을 그대로 두세요

### 3단계: 결과 확인 (1초)

"예측하기" 버튼 클릭!

**결과 예시:**

```
⏱️  예상 시간: 1642분 (약 27시간)
📊 신뢰 구간: 1011분 ~ 2273분
🎯 신뢰도: Medium (±25%)
🧠 데이터 출처: AI 앙상블 예측

단계별 시간:
  📥 데이터 로딩: 328분
  🔧 전처리: 493분
  ⚙️ 분석 실행: 821분
```

### 4단계: 해석하기

**예상 시간**: 가장 가능성 높은 시간
- "보통 이 정도 걸려요"

**신뢰 구간**: 최소~최대 범위
- "빠르면 1011분, 느리면 2273분"
- 70% 확률로 이 범위 안에 들어옴

**신뢰도**: 예측의 정확도
- High (±20%): 매우 정확
- Medium (±25%): 정확
- Low (±40%): 참고용

**데이터 출처**: 예측 방법
- 🧠 AI 앙상블: 가장 정확 (권장)
- ✅ 실제 데이터 기반: 정확
- ⚠️ 복잡도 추정: 참고용

## 🔧 방법 2: Python으로 더 정확하게 (선택)

### 준비 단계 (1번만 실행)

#### 1. Python 설치 확인

```bash
python --version
```

Python 3.8 이상이어야 합니다.

**없으시다면?**  
→ https://python.org 에서 다운로드

#### 2. 필요한 프로그램 설치

```bash
pip install -r requirements.txt
```

**설치되는 것들:**
- pandas: 데이터 처리
- numpy: 수치 계산
- scikit-learn: 머신러닝
- openml: 벤치마크 데이터

**예상 시간**: 1-3분

#### 3. 벤치마크 데이터 수집

```bash
python data_fetcher.py
```

**무슨 일이 일어나나요?**
- scitime 방식으로 320개 벤치마크 생성
- OpenML에서 실제 데이터 수집
- `benchmark_data.json` 파일 생성

**예상 시간**: 5-10분

**결과:**
```
============================================================
벤치마크 데이터 수집 완료!
============================================================
✅ 320개의 scitime 기반 벤치마크 생성
✅ OpenML에서 0-50개 추가 수집
💾 benchmark_data.json 저장 완료
```

#### 4. ML 모델 학습 (자동)

```bash
python ml_predictor.py
```

**무슨 일이 일어나나요?**
- 랜덤 포레스트 모델 학습
- `ml_model.pkl` 파일 생성

**예상 시간**: 10-30초

**결과:**
```
============================================================
머신러닝 모델 학습 시작
============================================================
📊 320개의 벤치마크 데이터 로드
🧠 랜덤 포레스트 모델 학습 중...
✅ 학습 완료!
   R² Score: 0.977
   평균 오차: ±67.5%
💾 모델 저장 완료
```

### 사용 단계

이제 `index.html`을 열면 **자동으로 ML 모델을 사용**합니다!

더 정확한 예측을 즐기세요 🎉

## 🐛 문제 해결

### 문제 1: "⚠️ 복잡도 추정" 메시지

**의미**: 비슷한 케이스를 찾지 못함

**해결법:**
1. Python 방법으로 ML 모델 학습 (권장)
2. 다른 알고리즘 선택해보기
3. 참고용으로만 사용

### 문제 2: 예측 시간이 너무 길어요

**원인:** 데이터가 크거나 알고리즘이 복잡

**해결법:**
1. 샘플링: 데이터의 10%만 사용
2. 하드웨어 업그레이드
3. 더 간단한 알고리즘 선택
4. 열 개수 줄이기

### 문제 3: Python 실행 오류

**오류: "pip 명령을 찾을 수 없습니다"**

```bash
python -m pip install -r requirements.txt
```

**오류: "UnicodeEncodeError"**

```bash
# Windows
$env:PYTHONIOENCODING='utf-8'; python data_fetcher.py

# Mac/Linux
PYTHONIOENCODING=utf-8 python data_fetcher.py
```

### 문제 4: 웹 페이지가 안 열려요

**해결법:**
1. 파일 경로 확인 (한글, 특수문자 주의)
2. 다른 브라우저로 시도
3. 파일을 바탕화면으로 복사 후 재시도
