# 데이터 분석 시간 예측 서비스 ⏱️

> **"이 분석, 얼마나 걸릴까?"**  
> 30초만 입력하면 1초 안에 예측 결과를 알려드립니다!

---

## 💡 이게 뭔가요?

데이터 분석을 **시작하기 전에**, 얼마나 걸릴지 미리 알려주는 서비스입니다.

---

## 😫 이런 경험 있으신가요?

### 실제 데이터 분석 현장의 고민

**📚 팀 프로젝트 할 때:**
- 카페에서 팀플 중 "이거 30분 안에 끝날까?"
- 시간이 더 걸릴 것 같아서 **중간에 끊고 처음부터 다시**
- 한정된 시간 안에 완료될지 몰라서 불안함...

**💻 피시방/노트북에서:**
- "언제 끝날지 몰라서 피시방에 무기한 대기..."
- "2시간 결제했는데 3시간 걸리면 어떡하지?"
- 자리 떠날 수도 없고, 계속 지켜봐야 함

**🌙 밤에 돌릴 때:**
- "자기 전에 돌려놓고 자야지..."
- 아침에 일어나보니 **오류로 중단됨** 😭
- 몇 시간을 기다렸는데 허무하게...

**⏰ 시간 계획 세울 때:**
- "점심 먹고 오면 끝날까?"
- "퇴근 전에 결과 나올까?"
- "방법 A vs B, 뭐가 더 빠를까?"

### 💔 가장 답답한 순간

```
돌려놓고 → 기다림 → 시간 더 걸림 → 끊음 → 처음부터 다시...
```

**이 무한 반복, 정말 지치죠?**

## 🧠 어떻게 예측하나요?

### 3가지 방법을 결합한 AI 앙상블

```
🧠 최종 예측 = 벤치마크(50%) + ML 모델(40%) + 복잡도(10%)
```

**1. 벤치마크 방식**
- 비슷한 케이스를 과거 데이터에서 찾기
- 가장 신뢰할 수 있는 방법

**2. ML 모델 (scitime 방식)**
- 랜덤 포레스트로 패턴 학습
- 없는 알고리즘도 예측 가능

**3. 복잡도 방식**
- 알고리즘 이론으로 계산
- 안전망 역할

### 정확도

| 방법 | 정확도 | 언제 사용? |
|------|--------|----------|
| 🧠 AI 앙상블 | ±20-25% | ML 모델 있을 때 (권장) |
| ✅ 벤치마크만 | ±20-30% | 비슷한 케이스 많을 때 |
| ⚠️ 복잡도만 | ±40% | 데이터 부족할 때 |

## 🔧 기술 정보 (개발자용)

### 파일 구조

```
porject12/
├── index.html              # 웹 인터페이스 (메인)
├── ml_predictor.py        # ML 모델 (scitime 방식)
├── predictor.py           # 예측 엔진 (앙상블)
├── data_fetcher.py        # 데이터 수집
├── benchmark_data.json    # 벤치마크 DB (자동 생성)
├── ml_model.pkl           # 학습된 모델 (자동 생성)
└── docs/                  # 문서
    ├── service-plan.md    # 서비스 계획
    ├── USAGE-GUIDE.md     # 사용 가이드
    ├── ML-MODEL-GUIDE.md  # ML 기술 문서
    └── IMPLEMENTATION-RESULT.md  # 구현 결과
```

### 기술 스택

- **Frontend**: HTML5, CSS3, JavaScript (순수)
- **Backend**: Python 3.8+
- **AI/ML**: scikit-learn (Random Forest)
- **데이터**: scitime + OpenML 벤치마크

### 성능

- **예측 속도**: 0.03초
- **ML 학습**: 10-30초
- **R² Score**: 0.977 (매우 높음)
- **모델 크기**: 5-10MB
